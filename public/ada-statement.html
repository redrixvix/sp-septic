Accessibility Statement

Last reviewed: April 6, 2026  ·  Next scheduled review: April 2027

S&P Septic & Excavating Inc. is committed to ensuring that our website is accessible to all users, including people with disabilities. We actively work to maintain and improve the accessibility of spseptic.com in accordance with the Web Content Accessibility Guidelines (WCAG) 2.1, Level AA, and applicable provisions of the Americans with Disabilities Act (ADA).

Accessibility Features We Provide

Perceivable

  • All meaningful images include descriptive alt text; decorative images use empty alt="".
  • Color contrast meets WCAG 2.1 AA standards (4.5:1 for normal text, 3:1 for large/bold text) throughout the site.
  • Content is not conveyed by color alone — text labels, icons, and patterns supplement color information.
  • The site respects the operating system prefers-reduced-motion setting — continuous animations pause or disable for users who request it.

Operable

  • A skip navigation link ("Skip to main content") appears on first keyboard tab and allows users to bypass repetitive navigation.
  • Full keyboard navigation support — all interactive elements (menus, forms, buttons, links) are reachable and operable via keyboard.
  • Focus indicators are visible, styled with a high-contrast outline, and never hidden.
  • The <html lang="en"> attribute is set on every page.
  • No content flashes or strobing animations are used on this site.
  • The Google Maps iframe on our Contact page includes a descriptive title attribute.
  • The mobile navigation menu is labelled with role="navigation" and aria-label landmarks.

Understandable

  • All form inputs have visible, programmatically associated <label> elements.
  • Required fields are marked with a visual asterisk and the required attribute.
  • Form error messages are announced to screen readers via role="alert" and aria-live="assertive".
  • Form success confirmations are announced via role="status" and aria-live="polite".
  • Form fields include appropriate autocomplete attributes to support browser autofill and assistive technology.
  • Language is set to English (lang="en") on every page.

Robust

  • The site is built with semantic HTML5 elements (<header>, <nav>, <main>, <footer>, <section>, <article>) used correctly.
  • ARIA landmarks (role="navigation", role="status", role="alert") are used appropriately and sparingly — not as a substitute for semantic HTML.
  • HTML is validated through build-time linting to catch unclosed tags and nesting errors.

Known Limitations

As of the date of this statement, the following areas are known to have limited accessibility:

  • Gallery page images — The Project Gallery currently uses placeholder photography (camera icon + "Photo coming soon"). When real project photos are added, they will include descriptive alt text specific to each project. Until then, these placeholders are marked with alt="" and are decorative.
  • Embedded videos — If video content is added to the site in the future, captions or a full text transcript will be provided in compliance with WCAG 2.1 Success Criterion 1.2.2.
  • Third-party content — The Google Maps embed is provided by Google and loads within an iframe. We cannot control the accessibility of Google's map interface, but the surrounding page content meets WCAG 2.1 AA standards.

Report an Accessibility Barrier

If you encounter any content, feature, or page on this website that you cannot access due to a disability, or if you believe any part of this site is not accessible, please contact us and we will make reasonable efforts to address the issue.

Enforcement & Legal Framework

This statement is prepared in accordance with the Americans with Disabilities Act (ADA), Title III, and the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA. The U.S. Department of Justice has affirmed that websites are covered under ADA Title III as "places of public accommodation."

In addition to ADA Title III, this site aims to conform to Ohio's civil rights protections for persons with disabilities.
